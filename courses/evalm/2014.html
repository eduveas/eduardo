707.031: Evaluation Methodology

This specialized course concentrates on evaluation methodologies for research and innovation. The course applies a user centric design (UCD) methodology. At each stage, appropriate methodologies will be introduced with a human perspective. The course will address conventional interfaces (e.g., visualization) and also frontier, less conventional methods (e.g., Augmented Reality, 3D User Interfaces)

The student taking this course will learn through experience to:

  • incorporate evaluation methodologies in the research process
  • understand which methodology is appropriate for a particular interface and research question
  • understand how to use advanced technology (eye-tracking, BCI) for evaluation purposes
  • understand and critique scientific text involving evaluation methodologies

More importantly, from experience, the student will learn how formulate research questions and hypotheses, how build evaluations to test those hypotheses, how to record and collect evidence, analyze it, derive knowledge from it, and report findings from evaluations.

Invited Talks

Validating Unconventional Interfaces
02.12.2014 @ 13:00 in the HS i1, Inffeldgasse 18, EG (HSEG058J)

Abstract:

With the popularization of 3D user interfaces driven forward by in particular 3D consoles like the Wii or Kinect, unconventional interfaces have found entrance in the daily life of many a user. Yet designing and, in particular, validating these kinds of interfaces is not straightforward. Within this talk, a design approach will be introduced that can be applied when designing and validating unconventional interfaces. The design approach, called human potential driven design, deploys the full potential of the human body to perceive sensory information and control an application. It follows a strongly human-factors driven methodology that pinpoints well specific issues that need to be carefully considered through the iterative stages of analysis, conceptualization, design and validation of an interface. Doing so, common pitfalls and lessons learned will be intertwined while illustrating the approach through a range of unconventional interfaces from different domains, including mobile technology, augmented and virtual reality. Next to visual interfaces, a particular focus will be put on stimulating all senses, by looking closely at tactile, haptic and multisensory interfaces.

Grading

20% participation 30% moderation topic 50% project topic

The course is project based. For each class students take turns to moderate and take part in participatory design. For the course, the student will:

  • choose a project topic to design and evaluate from concept to prototype
  • choose a project topic to moderate (play the role of user for peer projects)

Projects

Recommendations while reading:

A recommender system delivers a list of articles and media related to a text, such as a wikipedia page. There is a minimal format for each recommendation item, which can be extended by the authors. The intention of this project is to attract attention to recommended items. It would be useful to see what part of the text recommended items are related to.

Recommendations while writing

a recommender system delivers a list of scientific articles related to a piece of text, a keyword or keyphrase. The intention of this project is to introduce recommendations during writing (content creatiion). What kind of interfaces would be appropriate?.

Augmented recommendations

this project investigates interfaces to deliver recommendations to personal devices. The goal is to show recommendations related to a physical object (e.g., written text on a sheet of paper) that is captured with the personal device. Any sort of personal device is allowed (smartphone, glasses, handheld projector). Comparison of interfaces for different technologies are encouraged.

Visual augmentation with instructions

this project is intended to propose and evaluate methods to show instructions augmented onto real world objects. For example, we would like to guide a person in the preparation of a Cafe Moka, from disassembly of the Coffee pot to brewing the coffee.

BCI control of appliance:

this project is intended to evaluate methods to control a simple appliance using a EEG (BCI) or another bio-signal. The project should explore how to select the appliance and how to choose control modes and execute them.

What this course is NOT

This course is not an introduction to HCI nor to visualization. For general courses on the topics covered visit:

  • InfoVis (Keith Andrews)
  • HCI (Keith Andrews)
  • Visual Analytics (Marc Streit)
  • AKCG (Denis Kalkofen)
  • Augmented Reality (Dieter Schmalstieg)

This course will not deal with web usability, as that topic is extensively covered by Keith Andrews course on Information Architecture and Web Usability. Instead the course concentrates on providing a solid background on the different evaluation methods as applied in a UCD continuous evaluation process. This course is no replacement for an advanced statistics lecture. Descriptive and inferential statistics methods will be used to guide the preparation of data, the performance of appropriate statistical tests, their interpretation and reporting of results.

Who should attend this course

This course is particularly suited for post-graduate students (Msc.) and researchers (PhD.) investigating or developing novel metaphors to interact with computers and machines. It is also suited for researchers trying to understand the user behaviour and how it is influenced by technology. This includes psychology majors as well as educators. Students in the final year of their Bak. may benefit from this course too.
